# 走访详情 Walk Visit Detail PRD

## 需求背景

### 痛点
- **问题现象**：客户经理需要查看每条走访的完整详情，包括客户信息、走访现场信息、协同走访情况
- **发生频率**：高
- **当前 workaround**：通过纸质记录或线下文档查询

### 业务目标
- **量化指标**：详情页加载 < 500ms
- **目标期限**：持续可用

### 涉及系统/模块
- **模块名称**：自主走访 - 走访详情
- **变更类型**：新增
- **对接接口**：暂无（Mock 数据，复用 WalkVisitList 的 mockWalkVisits）

---

## 用户故事

### 故事1
- **角色**：客户经理
- **功能**：从走访列表点击卡片进入详情，查看走访完整记录
- **收益**：集中查看客户基本信息、走访现场信息
- **验收条件**：跳转 `/walk-visit/:walkVisitId` 后展示两段式详情

### 故事2
- **角色**：客户经理
- **功能**：点击"走访地址"后的"查看>"查看地图
- **收益**：快速定位走访地点
- **验收条件**：点击"查看>"弹出占位提示（地图功能开发中）

### 故事3
- **角色**：客户经理
- **功能**：点击"历史走访信息"按钮查看历史走访
- **收益**：回顾同一客户的全部走访记录
- **验收条件**：点击底部"历史走访信息"弹出占位提示（功能开发中）

## 页面结构

### 路由信息
- **路由路径** - 类型：文本；必填：是；示例：`/walk-visit/:walkVisitId`
- **页面标题** - 类型：文本；必填：是；示例：`走访详情`
- **访问权限** - 类型：枚举（登录）；描述：客户经理

### 布局结构
- **布局类型** - 类型：单栏（分两段卡片）
- **区域-顶部** - 返回按钮 + 标题
- **区域-客户基本信息** - 白底卡片（字段：客户名称、走访地址、派单类型、走访日期、走访人、走访人号码）
- **区域-走访信息** - 灰底块（标题 + 状态胶囊；字段：走访类型、联系人、联系电话、岗位、部门、走访内容、协同走访；条件展开：协同走访人员姓名、协同走访人员走访动作评价 或 是否蓝海客户）
- **区域-底部按钮** - 全宽"历史走访信息"按钮（fixed）

## 验收标准

### 正常流程
- [ ] **操作**：从走访列表点击卡片 → **预期**：跳转 `/walk-visit/:walkVisitId`
- [ ] **操作**：查看详情页 → **预期**：展示两段卡片（白底基本信息 + 灰底走访信息）
- [ ] **操作**：点击走访地址"查看>" → **预期**：弹出占位提示"地图查看功能开发中"
- [ ] **操作**：点击走访人号码 → **预期**：触发电话拨号
- [ ] **操作**：协同走访=无时查看详情 → **预期**：底部显示"是否蓝海客户"

### 异常流程
- [ ] **操作**：协同走访=有时查看详情 → **预期**：显示"协同走访人员姓名"和"协同走访人员走访动作评价"，不显示"是否蓝海客户"
- [ ] **操作**：点击底部"历史走访信息" → **预期**：弹出占位提示"历史走访信息功能开发中"
